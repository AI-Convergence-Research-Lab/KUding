# KUding

KUding은 **대학 프로그래밍 수업을 위한 시험 중심 온라인 코딩 평가 플랫폼**이다.  
중간·기말 시험 환경에 맞춰 설계되었으며, 문제 단위 점수 기반 자동 채점을 핵심으로 한다.

---

## 주요 특징

- 시험 생성 및 상태 관리
    
- 시험별 문제 구성 및 문제별 배점 설정
    
- 문제 단위 점수 기반 자동 채점
    
- 테스트케이스 균등 비율 평가
    
- 문제별 다중 제출 및 최고 점수 반영
    
- 그룹 단위 시험 대상 관리
    
- 문제 연동 Q&A 및 기본 커뮤니티 기능
    

---

## 점수 모델

- 점수의 공식 단위는 **문제**
    
- 문제 점수 = 문제 배점 × (통과 테스트케이스 수 / 전체 테스트케이스 수)
    
- 시험 점수 = 문제 점수 합
    
- 최종 점수만 저장 (집계 결과)

## 기술 스택

|구분|기술|
|---|---|
|Frontend|React 18|
|Frontend 언어|TypeScript|
|서버 상태 관리|TanStack Query v5|
|로컬 상태 관리|Zustand|
|라우팅|React Router v6|
|UI 라이브러리|MUI v6 또는 Ant Design v5|
|빌드 도구|Vite v5|
|Backend|NestJS (Node.js, TypeScript)|
|ORM|Prisma|
|Database|MySQL 또는 PostgreSQL|
|인증|JWT|
|채점 서버|Python + Docker|
|배포|단일 Node 서버|

---

## 아키텍처 개요

- 개발 시
    
    - Frontend(Vite)와 Backend(NestJS)를 분리 실행
        
- 배포 시
    
    - React를 빌드하여 NestJS가 정적 파일과 API를 함께 서빙
        
- 서버는 **하나의 NestJS 서버**로 운영

## 프로젝트 상태

- ERD 및 점수 모델 설계 완료
    
- 기술 스택 확정
    
- 구현 진행 중
    

---

## 설계 원칙

- 계산 가능한 점수는 저장하지 않음
    
- 원천 데이터와 집계 결과를 명확히 분리
    
- 시험 운영 흐름을 데이터 구조에 그대로 반영
    
- 단기 개발이지만 장기 사용을 고려한 구조
